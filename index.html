Массивы

Массив – структура данных, которая предназначена для хранения пронумерованных значений и предлагает дополнительные методы для удобного манипулирования такой коллекцией.

  • Проще всего массив представить в виде полки, на которой стоят несколько предметов. Каждый предмет - элемент массива. Только в случае с JS, этими предметами могут быть ещё и другие полки. Двумерных массивов в JS нет, но такая возможность легко реализуется с помощью вложенных друг в друга массивов.

Объявление массива

let array = ['Вячеслав', 'Марк', 'Анна', 'Елена', 'Анна', ...];

Массивы в JS не имеют ограничений по размеру (кроме, может быть, ограничений памяти) и по типу вмещаемых ими данных (т.е. можно затолкать туда и строку, и число, и объекты, и функции и т. д.)

Свойство length

console.log(array.length); // выведет длину массива

Как ни странно, но два пустых массива в JS не являются одним и тем же значением, поэтому

console.log([] === []); // вернёт false

При этом это не точно так же, как и с NaN. Если мы возьмём одну переменную, которой присвоен массив и сравним её с самой собой, 

console.log(array === array); // => true

Свойство length

Поэтому проверить, является ли массив пустым можно вот так:

			if (array.length === 0) {
	something();
}

А ещё со свойством length можно играться и этим самым совершать определённые непотребности с нашим массивом (но мы этим заниматься не будем). Это же свойство есть и у строк (я говорил об этом на предыдущем занятии)

Методы массива

У массивов много мощных методов, которые позволяют делать с ним практически всё, что угодно.

.push() // Добавляет значение в массив (в конец)
.pop() // Удаляет значение из массива, возвращает удалённое
значение (из конца)
.shift() и .unshift() // Действуют точно также, только
добавляют и удаляют из начала массива

Очень много вкусных и необходимых методов можно найти на https://learn.javascript.ru/array-methods

Область видимости

let a = 13;
let b = 15;
function spam() {
	let a = 10; // a внутри области видимости функции spam
	console.log(a); // 10
	b = a + 2; // b не объявлена в функции
	return a; // всё так же 10
}
let eggs = spam(); // 10
console.log(a, eggs, b); // 13 10 12

Hoisting

Hoisting (поднятие) - это когда объявления функций и переменных поднимаются в начало кода внутри области видимости.

iAmTotallyCoolFunction(); // => "Yes, I am!"

function iAmTotallyCoolFunction() {
	console.log("Yes, I am!");
}

И это сработает

Но предпочтительно всегда писать объявления сверху!

Замыкания

Замыкание - внешние переменные для функции.

var phrase = 'Апути!';
function say() {
	let text = phrase
	function go() {
		console.log( text ); // найдёт переменную снаружи
	}
	return go;
}
let a = say();
a(); // 'Апути!'
phrase = 'Нипутю!';
let b = say();
a(); // 'Апути!'
b(); // 'Нипутю!'

Коллбэки и таймеры

и стрелочные функции

setTimeout(() => { console.log('In a second!') }, 1000)
setInterval(() => { console.log('another second!') }, 1000)

Функции-параметры у setTimeout и setInterval выполняются асинхронно, поэтому "In a second!" и первое "another second!" выведутся (почти) одновременно.

У стрелочных функций нет своей локальной области видимости, поэтому их удобно вызывать в качестве функций обратного вызова

Крутые методы массивов

.filter(callback(item, index, arr)) // фильтрует массив
.sort(callback(item, index, arr)) // сортирует массив
.map(callback(item, index, arr)) // совершает действие над каждым элементом
.reduce(callback(item, index, arr)) // позволяет "усреднить" массив
